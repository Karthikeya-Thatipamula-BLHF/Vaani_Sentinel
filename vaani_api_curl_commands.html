üîê Authentication

POST Login

Purpose: Authenticate and get JWT token
Input: Username and password
Output: JWT access token, token type, expiry time
curl -X POST "https://vaani-sentinel-gs6x.onrender.com/api/v1/auth/login" \ -H "Content-Type: application/json" \ -d '{ "username": "admin", "password": "secret" }'

GET Get Current User Info

Purpose: Get information about the currently authenticated user
Input: JWT token in header
Output: User ID, username, role
curl -X GET "https://vaani-sentinel-gs6x.onrender.com/api/v1/auth/me" \ -H "Authorization: Bearer YOUR_TOKEN_HERE"

GET Validate Token

Purpose: Check if JWT token is valid
Input: JWT token in header
Output: Validation status and user info
curl -X GET "https://vaani-sentinel-gs6x.onrender.com/api/v1/auth/validate" \ -H "Authorization: Bearer YOUR_TOKEN_HERE"

POST Logout

Purpose: Logout current user session
Input: JWT token in header
Output: Logout confirmation message
curl -X POST "https://vaani-sentinel-gs6x.onrender.com/api/v1/auth/logout" \ -H "Authorization: Bearer YOUR_TOKEN_HERE"

üîß Testing with Postman

Step-by-step guide:

  1. Import to Postman: Copy any curl command and use "Import" ‚Üí "Raw text" in Postman
  2. Get Token: First run the login endpoint to get your JWT token
  3. Set Authorization: Copy the access_token from login response
  4. Add to Headers: Add "Authorization: Bearer YOUR_TOKEN" to other requests
  5. Test Endpoints: Start with health check, then login, then other endpoints

üìã Common Response Codes

  • 200: Success - Request completed successfully
  • 400: Bad Request - Invalid input data
  • 401: Unauthorized - Invalid or missing JWT token
  • 404: Not Found - Resource doesn't exist
  • 500: Internal Server Error - Server-side issue

üöÄ Quick Start Example
